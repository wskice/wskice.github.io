迭代器

记录每一次访问集合元素的位置

官方解释：

迭代器（Iterator）有时又称游标（cursor）是一个可以记住遍历的位置的对象。迭代器对象从集合的第一个元素开始访问，直到所有的元素被访问完结束。迭代器只能往前不会后退。

使用方式：通过iter函数获取可迭代对象的迭代器，对迭代器使用next方法访问集合元素里的元素

可以迭代的数据类型：

列表，字典，元组，字符串

不可迭代数据类型：

数字

StopIteration异常

此异常在使用next方法访问超过集合对象下标的元素时的报错，其实是一种迭代结束的标志

自定义“可迭代对象”

在定义对象时可以自定义对象是否能迭代

方法：在创建对象的类中，通过__iter__方法设置迭代器

如果查看对象是否可以迭代

内部原因：其实，使用iter()获取可迭代对象的 迭代器 时，会自动调用这个对象上的__iter__方法，并且这个方法返回迭代器。

生成器：特殊的迭代器

在python中，这种一边循环一遍计算的机制，称为生成器：generator。

什么是生成器：

通过列表生成式，可以直接创建一个列表。但受内存限制，列表容量肯定是有限的。数量庞大的列表会占用很大的存储空间，如果我们仅仅需要访问前面几个元素，那后面绝大多数元素占用的空间都白白蓝菲类。所以，如果列表元素可以按照某种算法推算出来，那我们就可以在循的过程中不断推算出后续的元素了，这样就不必创建完整的list(列表)，从而节省大量的时间和内存。

在python中，这种一边循环一遍计算的机制，称为生成器：generator。

大白话：通过算法优化列表，使列表只在读取的时候进行生成并访问，从而节省存储空间。(当然算法是啥的就不用深究了，python自带的一些奇奇怪怪的方式吧)

创建生成器方式1:

只需要吧一个列表生成式的[]改成()

创建 L 和 G 的区别仅在于最外层的 [ ] 和 ( ) ， L 是一个列表，而 G 是一个生成器。我们可以直接打印出列表L的每一个元素，而对于生成器G，我们可以按照迭代器的使用方法来使用，即可以通过next()函数、for循环、list()等方法使用。

__getattr__和__getattribute__方法

1. __getattr__

  • 在定义类的时候，设置此方法后，实例访问的属性是 类原本的属性 时，会调用此方法，并返回此方法的返回值。
  • 使用实例直接访问实例存在的实例属性时,不会调用__getattr__方法

有两个参数:

  • self指向创建出来的实例对象
  • item为返回实例对象的属性名

2. __getattribute__

__getattribute__仅在新式类中可用，重写__getattrbute__方法对类实例的每个属性访问都有效。

  • self指向创建出来的实例对象
  • item为返回实例对象的属性名

3. 同时定义时

__getattr__方法不会被调用，只会调用__getattribute__

描述符(descriptor)

如果一个 类 有个 属性 是 另个类 的 实例化对象 ，那么这个属性（也就是那个实例化对象)，就是一个描述符。

描述符就是那个 另一个类 ，也是一个普通的类

描述符中的设置：

__get__():

此方法是描述符中的方法，在实例化对象后，对象的属性被 访问 时将调用此方法，返回此方法返回的数据（默认不写也可以）

一共有三个参数：(self,instance,owner)

  • self指向实例对象的属性
  • instance 指向 调用描述符的对象（可通过id来获取地址）
  • owner 是此描述符所在类的
__set__():

此方法是描述符中的方法，在实例化对象后，对象的属性被 赋值 时调用此方法，返回此方法返回的数据(必须得有返回值)

一共有三个参数：(self,instance,value)

  • self指向实例对象的属性
  • instance 指向 调用描述符的对象（可通过id来获取地址）
  • value是实例化对象的属性 被 赋予的 新值
__delete__():

此方法是描述符中的方法，在实例化对象后，对象的属性被 删除 时调用此方法

  • self指向实例对象的属性
  • instance是此描述符所在的类及地址（可通过id来获取地址）
