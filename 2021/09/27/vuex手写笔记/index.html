vuex

介绍

vuex是vue项目中实现大范围数据共享的插件

使用Vuex的好处：

  • 数据的存取一步到位，不需要层层传递
  • 数据的流动非常清晰
  • 存储在VueX中的数据都是响应式的

基本使用

1.在/src/目录下创建stores文件夹，并在其下创建index.js

2.在index.js中导入Vue和Vuex

3.把Vuex挂在为Vue的插件

4.向外导出store的实例对象

5.在应用程序入口文件main.js导入stores，并挂载

VueX核心概念(原始和映射)

State

概念：State本质上就是Object对象 作用：用来存储全局共享的数据

基本使用：

  1. 在stores/index.js声明state（配置）

  2. 在组件中，仿问声明的state（使用）

    高级应用：

    1. 在stores/index.js声明state

    2. 从vuex中导入mapState映射辅助函数

      mapState为vuex插件里的辅助函数，如使用vuex里的数据较多时，为减少代码重复，方便使用vuex里的变量就可以使用mapState。

    3. 在computed计算属性中，把stores中声明的state映射到组件中

    4. 在组件中使用映射好的state

Mutations

Mutation本质上是 JavaScript 函数，专门用来变更 Store 中的数据。类似于vue中的methods 特点：想要修改 State 中的数据，只能调用 Mutation 方法！ 好处：能够确保修改来源的唯一性，方便调试和后期维护。

基本使用：

1.在组件中的button按钮上，给它添加一个click事件，

2.在methods中声明handlerAdd方法,并使用commit触发mutations中的方法

3.在stores/index.js声明mutations中的方法

高级使用：

1.在stores/index.js声明mutations中的方法

2.在组件中从vuex导入mapMutations映射辅助函数

3.在methods选项中，把 stores中声明的mutations的方法映射到组件中

4.在组件中，调用 映射到组件中的方法

WARNING

在项目开发中，为了保证 Store 中状态的每一次变化都是可追踪的，Vuex 规定： Mutation 必须是同步函数。 否则，vue-devtools 将无法正常追踪 Store 中数据的变化，这对大型项目的开发调试是灾难性的！

Actions

Action本质上是 JavaScript 函数，专门用来处理Vuex中的异步操作

基本使用：

1.在组件的button按钮上，给他添加一个click事件，

2.在methods中声明AsyncHandlerAdd方法，并使用dispath触发actions中的方法

3.在stores/index.js声明actions中的方法
